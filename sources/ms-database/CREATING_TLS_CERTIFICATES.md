# How to use SSL in MariaDB

First of all, in order to use SSL, you need to have a certificate that futhermore will be use to authenticate into the MariaDB server, instead of using username and password. Those certificates can be generate via an Certificate Authority (CA) or self-sigining.

In the microshop code a self-signed certificate (x509) is generated.

## Generating a certificate authority private key and a certificate

To create x509 certificates, you must generate a proper *private key* which is a RSA private key.
To generate the RSA *private key*, you need to execute the following prompt in the terminal:

```
openssl genrsa 2048 > ca-key.pem
```

With this code executed, a *private key* was generated.
Having the given *private key*, we should use the *ca-key.pem* in order to generate *ca.pem*

```
openssl req -new -x509 -nodes -days 365000 -key ca-key.pem -out ca.pem
```

So, we need a server private key and a self-signed certificate for our server.

## Generating a server private key and a self signed certificate

To generate a *private key* to our server, client, and another tools, and a *certificate request*(that will be furthermore used to receive the real certificate) we need to execute the following command:
```
openssl req -newkey rsa:2048 -days 365000 -nodes -keyout server-key.pem -out server-req.pem
```

After the last command, we need to remove the passphrase using the following command:
```
openssl rsa -in server-key.pem -out server-key.pem
```

After that, we need to have both the server's *private key* and the server's *certificate request* that are generated by executing the following command:

```
openssl x509 -req -in server-req.pem -days 365000 -CA ca.pem -CAkey ca-key.pem -set_serial 01 -out server-cert.pem
```

Now you have the following files:

- ca.pem
- ca-key.pem
- server-key.pem
- server-req.pem
- server-cert.pem

Whose only the needed keys for our database are:
- ca.pem
- server-key.pem
- server-cert.pem

## Verify the certificate

```
openssl verify -CAfile ca.pem server-cert.pem
```
